1. Inicialização e Configuração

  • git init – Inicia o repositório local.
  • git config --global user.name "nome" – Define o nome do usuário.
  • git config --global user.email "email" – Define o e-mail do usuário.
  • git config --list – Exibe todas as configurações.

2. Controle de Versão

  • git add . – Prepara todos os arquivos para commit.
  • git status – Exibe o estado atual dos arquivos.
  • git commit -m "mensagem" – Salva alterações no histórico.
  • git restore --staged arquivo – Desfaz a preparação.

3. Histórico e Visualização

  • git log – Mostra todo o histórico.
  • git log --oneline – Mostra o histórico resumido.
  • git checkout <hash> – Visualiza uma versão específica.
  • git checkout master – Volta para a branch principal.

4. Conectando ao GitHub

  • git remote add origin URL – Cria a conexão com repositório remoto.
  • git remote -v – Verifica a conexão com o remoto.
  • git pull origin main --allow-unrelated-histories – Sincroniza os históricos.
  • git push -u origin main – Envia para o GitHub.

5. Trabalhando com Branches

  • git clone URL – Clona o repositório remoto.
  • git pull origin branch – Baixa a branch remota.
  • git push origin branch – Envia alterações para a branch remota.
  • git fetch – Busca dados remotos sem aplicar.
  • git diff branch origin/branch – Compara local x remoto.

6. Comparando Visualmente

  • git show origin/branch:arquivo.html > temporario.html – Gera arquivo remoto temporário.
  • Usar VS Code: Select for Compare > Compare with Selected.
  • git merge origin/branch -m "mensagem" – Mescla se não houver conflitos.
